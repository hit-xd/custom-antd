# 步骤条 / Steps 使用规则

## 本规范包含 6 种形态：

1. 大尺寸水平方向步骤条（横向 32px）
2. 小尺寸水平方向步骤条（横向 24px）
3. 大尺寸竖向步骤条（竖向 32px）
4. 小尺寸竖向步骤条（竖向 24px）
5. 导航步骤条 - 无 icon
6. 导航步骤条 - 有 icon

# 5.5箭头导航步骤条样式 - 无 icon

组件由多个步骤模块组成，每个步骤包含一个主矩形区域。步骤之间的箭头感不是通过 margin gap 形成的，也不是把每个步骤整体做成 chevron 六边形，而是通过以下结构叠放实现：

1. 每个步骤都有一个 `134px × 48px` 的主矩形区域；
2. 非最后一个步骤右侧有一个 `12px` 宽的右三角箭头片；
3. 除第一个步骤外，每个步骤左侧有一个 `12px` 宽的 V 形覆盖片，用于压在前一个步骤的右箭头区域上；
4. 后一个步骤要覆盖在前一个步骤上方，形成连续的模块拼接效果。

## 整体结构

整体高度为 `48px`，宽度根据步骤数量自适应。

当前示例为 4 个步骤：

1. `1.已完成步骤`
2. `2.已完成步骤`
3. `3.进行中步骤`
4. `3.未开始步骤`

整体示例宽度为 `584px`。

尺寸规则：

* 步骤主矩形宽度：`134px`
* 步骤高度：`48px`
* 相邻步骤主矩形起始位置间隔：`150px`
* 右侧三角箭头片宽度：`12px`
* 左侧 V 形覆盖片宽度：`12px`
* 4 步时整体宽度：`584px`

步骤主矩形起始位置规则：

```ts
const STEP_BODY_WIDTH = 134;
const STEP_INTERVAL = 150;
const ARROW_WIDTH = 12;
const STEP_HEIGHT = 48;

const bodyX = index * STEP_INTERVAL;
```

4 个步骤的主矩形起始位置分别为：

```txt
第 1 步：x = 0
第 2 步：x = 150
第 3 步：x = 300
第 4 步：x = 450
```

注意：这里的 `150px` 不是步骤之间的空白间距，而是每个步骤主矩形的起始位置间隔。不要使用 `gap: 16px` 或 `margin-left: 16px` 把步骤分开。

## 模块几何结构

请按照以下方式绘制每个步骤模块。

### 第一个步骤

第一个步骤左侧是平直边，右侧有三角箭头片。

视觉路径可以理解为：

```txt
左上角 → 主矩形右上角 → 右箭头尖点 → 主矩形右下角 → 左下角 → 闭合
```

推荐 SVG path：

```ts
M x 0
H x+134
L x+146 24
L x+134 48
H x
Z
```

其中：

```ts
x = index * 150
```

### 中间步骤

中间步骤左侧有 V 形覆盖片，右侧有三角箭头片。

视觉路径可以理解为：

```txt
左侧覆盖片上端 → 主矩形右上角 → 右箭头尖点 → 主矩形右下角 → 左侧覆盖片下端 → 左侧 V 形尖点 → 闭合
```

推荐 SVG path：

```ts
M x-12 0
H x+134
L x+146 24
L x+134 48
H x-12
L x 24
Z
```

其中：

```ts
x = index * 150
```

### 最后一个步骤

最后一个步骤左侧有 V 形覆盖片，右侧是平直边，不再显示右箭头。

推荐 SVG path：

```ts
M x-12 0
H x+134
V 48
H x-12
L x 24
Z
```

其中：

```ts
x = index * 150
```

## 渲染顺序

必须按步骤顺序从左到右渲染，并且后一个步骤要覆盖在前一个步骤上方。

推荐实现方式：

```tsx
{steps.map((step, index) => (
  <path
    key={step.key}
    d={getStepPath(index)}
    fill={getStepBackground(step.status)}
  />
))}
```

不要把每个步骤做成彼此独立的卡片。
不要使用 `display: flex; gap: 16px` 来排列步骤。
不要让步骤之间出现明显空白。
不要把每个步骤都做成完整的 chevron 六边形。

正确效果应该是：后一项的左侧 V 形覆盖片叠在前一项右箭头区域上，形成连续拼接。

* 文字居中显示
* 无 icon
* 无圆点
* 无圆形数字
* 无额外边框
* 无普通横向连接线

### 进行中状态

用于当前正在进行的步骤。

样式：

* 背景色：`#9B7235`
* 文字颜色：`#FFFFFF`
* 文字居中显示
* 无 icon
* 无圆点
* 无圆形数字
* 无普通横向连接线

当前步骤的金色背景必须覆盖完整步骤形状，包括：

* 主矩形区域；
* 左侧 V 形覆盖片；
* 右侧三角箭头片。

不能只给 `134px × 48px` 的主矩形区域上色。
如果当前步骤是中间步骤，左侧覆盖片也必须是 `#9B7235`。
如果当前步骤不是最后一步，右侧三角箭头片也必须是 `#9B7235`。

### 未开始状态

* 文字垂直居中
* 文字区域宽度：`134px`
* 文字区域高度：`48px`
* 文字区域位置跟随主矩形起始位置，不要把文字放到整个箭头形状的几何中心

文字区域位置：

```ts
left = index * 150;
top = 0;
width = 134;
height = 48;
```

文字内容中的序号是普通文本的一部分，例如：

```txt
1.已完成步骤
2.已完成步骤
3.进行中步骤
3.未开始步骤
```

不要把序号单独提取成圆形数字。
不要生成步骤圆点。
不要生成 Ant Design Steps 的 `icon + title + tail` 结构。

## 推荐实现方式

推荐使用 SVG 实现整体步骤条。

外层 SVG：

```tsx
<svg
  width={computedWidth}
  height={48}
  viewBox={`0 0 ${computedWidth} 48`}
>
  {steps.map((step, index) => (
    <path
      key={step.key}
      d={getStepPath(index, steps.length)}
      fill={getStepFill(step.status)}
    />
  ))}

  {steps.map((step, index) => (
    <foreignObject
      key={`${step.key}-label`}
      x={index * 150}
      y={0}
      width={134}
      height={48}
    >
      <div className="private-arrow-step-label">
        {step.label}
      </div>
    </foreignObject>
  ))}
</svg>
```

路径生成规则：

```ts
const BODY_WIDTH = 134;
const STEP_INTERVAL = 150;
const ARROW_WIDTH = 12;
const HEIGHT = 48;
const MID_Y = 24;

function getStepPath(index: number, total: number) {
  const x = index * STEP_INTERVAL;

  if (total === 1) {
    return `M ${x} 0 H ${x + BODY_WIDTH} V ${HEIGHT} H ${x} Z`;
  }

  if (index === 0) {
    return `
      M ${x} 0
      H ${x + BODY_WIDTH}
      L ${x + BODY_WIDTH + ARROW_WIDTH} ${MID_Y}
      L ${x + BODY_WIDTH} ${HEIGHT}
      H ${x}
      Z
    `;
  }

  if (index === total - 1) {
    return `
      M ${x - ARROW_WIDTH} 0
      H ${x + BODY_WIDTH}
      V ${HEIGHT}
      H ${x - ARROW_WIDTH}
      L ${x} ${MID_Y}
      Z
    `;
  }

  return `
    M ${x - ARROW_WIDTH} 0
    H ${x + BODY_WIDTH}
    L ${x + BODY_WIDTH + ARROW_WIDTH} ${MID_Y}
    L ${x + BODY_WIDTH} ${HEIGHT}
    H ${x - ARROW_WIDTH}
    L ${x} ${MID_Y}
    Z
  `;
}
```

宽度计算规则：

```ts
const computedWidth = (steps.length - 1) * 150 + 134;
```

4 步时：

```ts
computedWidth = 3 * 150 + 134 = 584;
```

整体高度：`48px`
整体宽度：根据步骤数量自适应

单个步骤主矩形宽度：`134px`
单个步骤高度：`48px`

左侧 V 形覆盖片宽度：`12px`
相邻步骤主矩形起始位置间隔：`150px`

文字区域宽度：`134px`
文字区域高度：`48px`

## 禁止事项

禁止使用 Ant Design Steps 默认样式。
禁止生成圆点。
禁止生成圆形数字。
禁止生成普通横向连接线。
禁止把步骤做成普通按钮组。
禁止使用 Tabs、Segmented、ButtonGroup 替代。
禁止使用 `gap` 或 `margin` 把步骤分成多个独立块。
禁止把每个步骤做成独立 chevron 卡片。
禁止让步骤之间出现大面积空白。
禁止给每个步骤加圆角、阴影、卡片边框。
禁止让当前步骤只显示一个普通矩形背景。
禁止把当前步骤做成孤立的金色六边形。
禁止把文字放在整个箭头形状中心，文字必须居中在 `134px × 48px` 主矩形区域内。

## 验收标准

生成后必须满足：

1. 整体高度为 `48px`。
2. 4 步整体宽度为 `584px`。
3. 每个步骤主矩形宽度为 `134px`。
4. 每个步骤主矩形起始位置为 `0 / 150 / 300 / 450`。
5. 非最后一步右侧有 `12px` 三角箭头片。
6. 除第一步外，每一步左侧有 `12px` V 形覆盖片。
7. 步骤之间是连续叠放关系，不是独立按钮之间留白。
8. 当前步骤的金色背景覆盖主矩形、左侧覆盖片和右侧箭头片。
9. 页面中没有圆点、圆形数字、普通连接线。
10. 文字居中在主矩形区域内，而不是居中在整个箭头外轮廓内。
