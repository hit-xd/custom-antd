# 审批专用步骤条 / Approval Progress Steps 样式规范

结构包括：

- 纵向线；
- 已完成节点；
- 当前待处理节点；
- 已驳回节点；
- 已回退节点；
- 未开始节点。

样式要求：

- 左侧流程线宽度：`2px`
- 左侧流程线位于组件左侧，节点居中压在线上
- 已完成段流程线颜色：`#00C078`
- 未开始段流程线颜色：`#DADCDE`
- 异常节点后的流程线颜色：按业务状态处理，颜色 `#DADCDE`
- 流程线贯穿节点之间，不要断开
- 节点应覆盖在线条上方

### 3.2 节点状态图标

节点用于表达审批节点状态。

节点类型包括：

1. 已完成节点；
2. 待处理节点；
3. 已驳回节点；
4. 已回退节点；
5. 未开始节点。

节点统一位于左侧流程线上，节点中心点与流程线水平居中对齐。

### 3.3 右侧审批内容区域

右侧审批内容区域用于展示当前节点或历史节点的审批信息。

结构包括：

- 审批意见卡片；
- 驳回原因卡片；
- 附件信息；
- 右侧状态文案。

右侧内容区域与左侧流程线之间保持固定间距。

### 3.4 审批意见卡片

审批意见卡片用于展示审批意见文本。

示例文案：

```txt
同意申请 审批意见（有审批意见则展示）
```

卡片特点：

- 浅灰背景；
- 横向长条；
- 文案左对齐；
- 高度较低；
- 通常展示在已完成节点右侧下方。

### 3.5 驳回 / 回退原因卡片

驳回 / 回退原因卡片用于展示失败类流程信息。

示例文案：

```txt
驳回原因：缺少附件材料，请上传
附件：组件123.jpg
```

或：

```txt
驳回原因：缺少附件材料，请上传
```

卡片特点：

- 浅灰背景；
- 原因文案使用红色；
- 附件信息使用常规文字色；
- 带附件时卡片高度更高；
- 不带附件时卡片高度更低。

## 4. 状态规则

### 4.1 已完成状态

用于表示该审批节点已经完成或通过。

节点样式：

- 节点形态：圆形节点，内部为白色对勾
- 节点主色：`#00C078`
- 节点尺寸：建议 `14px × 14px`
- 对勾颜色：`#FFFFFF`
- 节点位于左侧流程线中心

流程线样式：

- 已完成节点之前或已完成链路段使用绿色流程线
- 已完成流程线颜色：`#00C078`

内容展示：

- 可展示审批意见卡片
- 审批意见文字使用默认正文色
- 不展示红色异常文案

### 4.2 待处理状态

用于表示当前节点正在等待处理。

节点样式：

- 节点形态：蓝色圆形节点，内部为白色状态点 / 省略点
- 节点主色：`#1874FF`
- 节点尺寸：建议 `14px × 14px`
- 内部图形颜色：`#FFFFFF`

右侧状态文案：

- 文案示例：`待处理`
- 文字颜色：`#1874FF`
- 字号：`14px`
- 字重：`Regular`
- 行高：`22px`
- 右侧对齐，位于当前节点横向对齐位置附近

内容展示：

- 可展示审批意见卡片
- 待处理节点不展示原因卡片

### 4.3 已驳回状态

用于表示审批节点被驳回。

节点样式：

- 节点形态：红色圆形节点，内部为白色叉号
- 节点主色：`#FF4D4F`
- 节点尺寸：建议 `14px × 14px`
- 内部叉号颜色：`#FFFFFF`

右侧状态文案：

- 文案示例：`已驳回`
- 文字颜色：`#FF4D4F`
- 字号：`14px`
- 字重：`Regular`
- 行高：`22px`
- 右侧对齐，位于驳回节点横向对齐位置附近

原因卡片：

- 必须支持展示驳回原因
- 原因文字颜色：`#FF4D4F`
- 原因文字字号：`14px`
- 原因文字字重：`Regular`
- 原因文字行高：`22px`

附件信息：

- 当存在附件时，在原因下方展示附件行
- 附件标题颜色：`rgba(0, 0, 0, 0.65)`
- 附件文件名颜色：`rgba(0, 0, 0, 0.65)`
- 附件图标颜色：`rgba(0, 0, 0, 0.45)`

### 4.4 已回退状态

用于表示审批流程被退回上一节点或退回补充材料。

节点样式：

- 节点形态：红色圆形节点，内部为白色回退 / 退回图标
- 节点主色：`#FF4D4F`
- 节点尺寸：建议 `14px × 14px`
- 内部图形颜色：`#FFFFFF`

右侧状态文案：

- 文案示例：`已回退`
- 文字颜色：`#FF4D4F`
- 字号：`14px`
- 字重：`Regular`
- 行高：`22px`

原因卡片：

- 支持展示回退 / 驳回原因
- 原因文字颜色：`#FF4D4F`
- 原因文字字号：`14px`
- 原因文字字重：`Regular`
- 原因文字行高：`22px`

### 4.5 未开始状态

用于表示尚未到达的审批节点。

节点样式：

- 节点形态：灰色空心 / 浅灰节点
- 节点主色：`#DADCDE`
- 节点尺寸：建议 `14px × 14px`
- 节点中心可保留白色或透明内芯
- 不展示文字
- 不展示状态文案
- 不展示内容卡片

流程线样式：

- 未开始段流程线颜色：`#DADCDE`

## 5. 尺寸 / 间距 / 颜色 / 交互规则

### 5.1 整体尺寸

当前设计稿示例宽度：`313px`  
组件整体高度：根据审批节点数量和内容卡片数量自适应  
组件背景：透明  
外层描边：无  
外层圆角：无

推荐布局：

- 左侧流程线区域宽度：`16px`
- 右侧内容区域起始位置：`24px`
- 右侧内容区域宽度：自适应，当前示例为 `289px`
- 左侧流程线中心位置：距离组件左侧约 `8px`
- 内容区域与流程线之间的水平间距：约 `16px`

### 5.2 流程线尺寸

流程线宽度：`2px`  
流程线颜色：

- 已完成段：`#00C078`
- 未开始段：`#DADCDE`
- 异常后续段：建议 `#DADCDE`

流程线应垂直贯穿节点之间。

节点应覆盖在流程线上方，避免流程线穿透节点图标。

### 5.3 节点尺寸

已完成节点：

- 尺寸：建议 `14px × 14px`
- 背景色：`#00C078`
- 内部图标：白色对勾

待处理节点：

- 尺寸：建议 `14px × 14px`
- 背景色：`#1874FF`
- 内部图标：白色状态点 / 省略点

已驳回节点：

- 尺寸：建议 `14px × 14px`
- 背景色：`#FF4D4F`
- 内部图标：白色叉号

已回退节点：

- 尺寸：建议 `14px × 14px`
- 背景色：`#FF4D4F`
- 内部图标：白色回退 / 退回符号

未开始节点：

- 尺寸：建议 `14px × 14px`
- 外圈颜色：`#DADCDE`
- 内部中心色：`#FFFFFF` 或透明
- 不展示图标

### 5.4 审批意见卡片尺寸

审批意见卡片用于展示常规审批意见。

样式规则：

- 卡片背景色：`#F4F6F8`
- 卡片宽度：自适应，当前示例为 `289px`
- 卡片最小高度：`38px`
- 卡片圆角：建议 `2px`
- 卡片内边距：建议 `8px 16px`
- 文案左对齐
- 文案垂直居中

审批意见文字：

- 颜色：`rgba(0, 0, 0, 0.85)`
- 字号：`14px`
- 字重：`Regular`
- 行高：`22px`

### 5.5 驳回 / 回退原因卡片尺寸

原因卡片用于展示驳回、回退等异常原因。

带附件时：

- 卡片背景色：`#F4F6F8`
- 卡片宽度：自适应，当前示例为 `289px`
- 卡片高度：建议 `64px`
- 卡片圆角：建议 `2px`
- 卡片内边距：建议 `10px 12px`

不带附件时：

- 卡片背景色：`#F4F6F8`
- 卡片宽度：自适应，当前示例为 `289px`
- 卡片高度：建议 `38px`
- 卡片圆角：建议 `2px`
- 卡片内边距：建议 `8px 12px`

原因文字：

- 颜色：`#FF4D4F`
- 字号：`14px`
- 字重：`Regular`
- 行高：`22px`

附件文字：

- 颜色：`rgba(0, 0, 0, 0.65)`
- 字号：`14px`
- 字重：`Regular`
- 行高：`22px`

附件图标：

- 尺寸：建议 `14px × 14px`
- 颜色：`rgba(0, 0, 0, 0.45)`
- 与附件文字间距：`4px`

### 5.6 状态文案样式

状态文案位于右侧内容区域右侧或当前节点横向位置附近，用于标识当前状态。

待处理：

- 文案：`待处理`
- 颜色：`#1874FF`

已驳回：

- 文案：`已驳回`
- 颜色：`#FF4D4F`

已回退：

- 文案：`已回退`
- 颜色：`#FF4D4F`

状态文案通用文字样式：

- 字号：`14px`
- 字重：`Regular`
- 行高：`22px`
- 水平对齐：右对齐
- 不应与内容卡片重叠

### 5.7 垂直间距规则

节点之间的垂直间距根据内容高度自适应。

建议规则：

- 相邻节点中心之间的最小距离：`62px`
- 审批意见卡片顶部距离对应节点：根据节点位置自适应，建议与视觉稿保持约 `44px`
- 原因卡片与状态节点之间的垂直间距：建议 `36px` 左右
- 若节点包含多行内容或附件，后续节点应整体下移，避免重叠
- 后续未开始节点保持均匀垂直分布

### 5.8 交互规则

步骤条默认用于展示审批进度，不作为导航切换控件。

交互规则：

- 节点默认不可点击；
- 审批意见卡片默认不可点击；
- 附件文件名可根据业务需要支持点击预览或下载；
- 鼠标悬停在附件文件名上时，可展示链接态；
- 鼠标悬停在普通节点或普通卡片上时，不改变样式；
- 组件不应因为 hover 改变节点尺寸或卡片尺寸；
- 若后续业务要求节点可点击查看详情，可额外增加点击态，不应影响默认展示态。

## 6. 注意事项

1. 左侧流程线必须保持连续。
   - 节点应覆盖在流程线上方。
   - 不要让流程线穿透节点图标。
   - 已完成段和未开始段颜色需要根据节点状态区分。

2. 审批意见卡片和原因卡片不要做成普通 Tooltip。
   - 它们是流程内容的一部分，应常驻展示。

3. 状态文案不要放进卡片内部。
   - `待处理`、`已驳回`、`已回退` 应作为右侧状态标签独立展示。

4. 驳回 / 回退原因必须使用红色文字。
   - 原因文字颜色统一为 `#FF4D4F`。
   - 附件行不使用红色，保持普通辅助文字色。

5. 未开始节点不展示内容卡片。
   - 仅展示灰色节点和未开始段流程线。

6. 如果审批意见内容为空，应支持隐藏审批意见卡片。
   - 但节点和流程线仍需正常展示。

7. 如果驳回 / 回退原因较长，应支持内容换行。
   - 卡片高度需要随内容自适应。
   - 不允许文字溢出卡片。

8. 如果存在附件，应支持多附件扩展。
   - 多附件可换行展示或以列表展示。
   - 附件图标与文件名需要保持同一行对齐。

9. 组件应支持不同审批流程数量。
    - 节点数量不应写死。
    - 内容卡片高度不应写死到单一场景。
    - 整体高度应根据节点数量和内容动态计算。
